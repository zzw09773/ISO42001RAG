# 前端「印刷報告書」視覺系統重構 — 設計文件

- 日期：2026-07-08
- 狀態：已核准（使用者於本日核准）
- 範圍：monitoring 儀表板（`/dashboard`）、稽核日誌搜尋（`/audit`）、md2html 文件頁與 INDEX 入口
- 明確排除：OpenWebUI、code-server、Keycloak、Jupyter（第三方現成品）；`RAG/*.py`（凍結範圍）

## 背景與目標

本專案為 ISO 42001 稽核標的。現有自製前端有兩套視覺（儀表板深藍卡片風、文件頁深藍 docbar 風），且儀表板含 emoji 狀態標記與漸層等裝飾。目標是建立一套「印刷報告書」視覺系統，讓三種受眾（外部稽核員、內部維運工程師、長官/管理層）在同一頁面由淺入深各取所需，並使任何時間點「另存 HTML／列印」即為完整、可辨識的稽核證據。

設計原則取自 `Claude-Design-Sys-Prompt.txt` 的設計指引（避免 AI slop：不濫用漸層、不用 emoji、不用圓角+左邊框色條堆疊、少即是多、先建立一致視覺系統再動手），不採用該文件中的工具協定（CDN、React、Tweaks 等，內網離線不適用）。

## 已確認的決策

| 決策點 | 結論 |
|---|---|
| 範圍 | 儀表板 + 文件頁兩者都做，共用視覺系統 |
| 改動深度 | 全面重構（資訊架構重排），但資料與判定邏輯不動 |
| 受眾 | 稽核員、工程師、長官三者兼顧，以分層結構解決 |
| 互動 | 保留既有 30 秒自動更新與 SSE 即時告警；不新增互動 JS |
| 視覺基調 | 印刷報告書風（黑白灰 + 單一深藍強調 + 狀態色僅用於狀態值） |
| IA 方案 | 單頁分層報告書（執行摘要 → A/B/C 章 → 附錄 + 純錨點目錄） |

## 1. 共用視覺系統

- 色彩 token：正文 `#1a1f2c`、輔助灰階、強調色維持 `#1e3a8a`（條文編號、連結、章節編號）。狀態色（紅/琥珀/綠）只著色於狀態值本身（文字或色點），不做大面積底色。
- 字型：`"Noto Sans TC","Microsoft JhengHei","PingFang TC"` 本機字型鏈；數值、時間戳、ID 用等寬字型；`font-variant-numeric: tabular-nums`。
- 結構語彙：無陰影、無漸層；細橫線與留白分區；表格細線框；章節採「編號 + 中文標題 + 英文小字」雙語小標。
- 狀態標記：全面去 emoji。`🟢 intact` → `● INTACT`（色點 + 文字）。色點維持 RGB 彩色（使用者確認：螢幕上用色不受限，列印為黑白印表機，語意由文字承載即可，不需為列印做灰階設計）。
- 列印：`@media print` 隱藏 live 指示與 SSE 狀態；目錄、表格、SVG 圖不裁切。
- token 區塊在 `scripts_md2html.py` 與 `dashboard_render.py` 各自內嵌（monitoring_addon 是獨立 Docker build context，不跨界共用檔案），兩處以相同註解標頭標明同屬一套視覺系統，修改時需同步。

## 2. 儀表板 `/dashboard` 資訊架構

```
報告頭    系統名稱 / 產生時間 / 資料視窗 / audit 鏈完整性 / live 狀態
目錄      純錨點連結（零 JS）
執行摘要  整體狀態（A/B/C 取最差，不以平均稀釋）＋ 業務目標 Hit Rate ≥ 0.90
          兩結論並列，各附一句判定依據 —— 長官一屏看完
第 A 章   運作健康：章頭固定「觀察問題 → ISO 條文 → 門檻/實際/判定」對照表，
          之後 KPI、每日趨勢圖、異常旗標
第 B 章   品質保證：同章頭結構，之後 Hit Rate 目標卡、V&V 基線快照
第 C 章   服務健康：同章頭結構，之後健康分數、近 24 小時延遲狀態格
          （維持現有進度條/格狀樣式，使用者指定保留）、可用率、完整性
第 D 章   告警（近 24 小時）：橫幅 + 明細表（SSE 即時插入）
附錄      方法論 / 門檻依據 / 原始量測值 / 安全控制
頁尾      audit dir、SSE 連線狀態
```

- 資料層（`dashboard_data.py`）、判定函式（`_dim_status_A/B/C`、`_compute_overall`）、SVG 圖表函式介面**不動**；本次為 render 層（HTML 結構 + CSS）重構。
- `#live-content` 依然包住全部可變區域，30 秒自動更新機制照舊。
- SSE 告警插列、pill 計數、hero 提示的 DOM 選擇器隨新結構調整，功能等價。
- 「思考過程」透明性內容（方法論、門檻依據、判定理由）保留並升格為正式附錄章節。

## 3. `/audit` 稽核日誌搜尋頁

套用同一視覺 token；檢索表單改為報告書式條件列；結果表用細線表格。查詢參數、後端行為、OpenWebUI 關聯功能不變。

## 4. 文件頁（scripts_md2html.py + INDEX）

- CSS 模板改為報告書風：深藍 docbar 改為白底細線報告頭，文件路徑與產生日期靠右。
- `INDEX.md`/`INDEX.html` 改為「文件總覽」：按治理、技術、證據分組的目錄表。
- 完成後重跑 `python3 scripts_md2html.py` 重新產生全部 HTML。

## 5. 資料流與錯誤處理

全部不變：路由、查詢參數、空資料 fallback 文案（如「尚未載入 V&V 報告」）照舊。
