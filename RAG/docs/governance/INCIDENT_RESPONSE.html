事件回應程序（Incident Response Procedure）

ISO 42001 A.8（事件管理）證據。 狀態：v1.1.0（2026-06-10 編製，SLA 與通報對象待單位核定）

1. 事件分類與來源

類別 偵測來源 範例
安全事件 audit log security_alert 事件 prompt 注入、越權嘗試、異常存取模式
品質事件 V&V fail、人工抽查 Hit Rate 跌破 0.90、引用條文不一致
完整性事件 稽核日誌雜湊鏈驗證 稽核日誌雜湊鏈斷裂
可用性事件 容器健康檢查、audit log fallback 軌跡 推論後端中斷、服務降級

事件紀錄以稽核日誌、CHANGELOG 與本文件指定的人工處置紀錄為準。

2. 回應流程

事件發現 → 系統管理者確認（分級）→ 調查 → 矯正/緩解 → 處置紀錄 → 結案簽核
  1. 確認：系統管理者於 SLA 內確認事件（見 §3），判定真實事件或誤報。
  2. 調查：以 audit log（actions 欄位含完整工作流軌跡）還原事件； critical 安全事件須通報稽核負責人。
  3. 矯正：依事件類別處置（重啟服務／調整控制／重新索引／回退版本—— 回退程序見 INTRANET_DEPLOYMENT_RUNBOOK.html 緊急回退章節）。
  4. 紀錄：每筆 warning 以上事件須留下處置紀錄 （調查 → 根因 → 矯正 → 處置決定），由稽核負責人簽核結案。
  5. 根因迴饋：若根因屬系統性缺陷，更新 AI_RISK_ASSESSMENT.md 風險登錄。

3. 回應 SLA（單位核定）

嚴重度 確認時限（建議） 處置時限（建議） 核定值
critical 4 小時 1 個工作日 ☐
